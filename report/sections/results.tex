\section{Discussion \& Conclusion}
\label{sec:discussion}

The results so far support the central motivation of this project: improving long-horizon manipulation robustness cannot be reduced to changing the policy architecture alone. In the full-episode baseline, ACT and SmolVLA achieve comparable performance when both are trained on the same monolithic demonstrations and executed without recovery. This indicates that the main limitation is not only the expressiveness of the low-level policy, but also the structure of the execution problem.

The monolithic formulation gives the policy no explicit opportunity to verify that an intermediate physical outcome has been achieved. If a local subtask fails, the policy can continue producing plausible actions even though the assumptions needed for those actions are false. For a long-horizon task, this creates a compounding failure problem: a small error early in the sequence can invalidate the rest of the episode.

Subtask decomposition addresses this issue by introducing intervention points. A grasp, lift, or placement behaviour can be evaluated before the system commits to the next step. This does not make the low-level policy perfect, but it changes how failures affect the global task. Instead of silently propagating through the remainder of the episode, a failure can be detected and handled by the execution system.

The sliced public dataset experiment shows that subtask boundaries are not sufficient by themselves. For recovery, the system must also be able to choose among alternative valid continuations. In the orange-picking task, this means being able to request a specific target orange. A dataset with a fixed object order does not provide the right supervision for this behaviour, because target identity is entangled with progress through the task. The manually generated dataset was therefore necessary to make relative-position instructions meaningful.

Local retry is the only true recovery mechanism: it returns to GRASP for the same target orange after a slip during LIFT or PLACE, preserving the original sub-goal. Target redirection is goal abandonment, not recovery: the orchestrator gives up on the current orange after repeated failures and attempts a different one. This prevents dead-end execution by finding an alternative path toward the global objective, but it does not recover the original sub-goal. The system handles local slips via retry and persistent failures via redirection, and neither substitutes for a reliably successful subtask policy.

The formal orchestrated evaluation shows that this execution structure is not yet sufficient to improve complete-task performance. The manually collected subtask model achieved 26 successes in 326 evaluated episodes, or 7.98\%, with an average of 1.32 oranges placed. This is substantially worse than the monolithic full-episode baselines, which reached 26\% for ACT and 27\% for SmolVLA. The result suggests that the benefits of explicit subtask monitoring are outweighed by the low reliability of the learned grasping behaviour, especially as the scene evolves after one or more oranges have already been placed.

Several limitations remain. First, the success checks use privileged simulator state. This is useful for isolating the recovery problem, but it would need to be replaced or approximated by perception-based success detection for deployment outside simulation. Second, the orchestrated evaluation was stopped after 326 of the planned 500 episodes, so the reported value should be treated as a partial evaluation rather than a final benchmark. Third, the merged model evaluation (N=41, preliminary) shows a lower GRASP success rate in later scene states than the handmade-only model, indicating that adding 414 autonomous episodes has not yet improved the identified bottleneck. Whether this result holds at the full planned 500-episode sample is pending.

Future work should complete the autonomous data generation cycle and evaluate whether the generated successful subtasks improve robustness, especially in out-of-distribution configurations. Completion of the merged model evaluation (currently N=41) will confirm whether the observed GRASP degradation persists, and whether the fair-comparison monolithic model (currently N=247, 17.41\% full completions) is representative of what the merged data can produce without language conditioning. A further extension would replace privileged success checks with visual or multimodal failure detection, making the execution framework less dependent on simulator-only information.

In conclusion, this project frames recovery as an execution-level requirement for VLA-based robotic manipulation. The work so far shows that monolithic full-episode imitation has limited robustness, that subtask decomposition creates the intervention points needed for recovery, and that target-conditioned subtasks are necessary for meaningful redirection after failure. At the same time, the current orchestrated result is worse than the monolithic baselines, showing that orchestration cannot compensate for an insufficiently reliable subtask policy. The autonomous data generation phase collected 414 subtask episodes using this execution structure, forming a merged training dataset of 1335 episodes. Preliminary evaluation of the resulting model (N=41) suggests that GRASP reliability did not improve with the added autonomous data; the full 500-episode evaluation will determine whether this finding holds.

\section{Introduction}
\label{sec:introduction}

Long-horizon robotic manipulation requires executing a sequence of physical interactions whose success depends on the evolving state of the world. A pick-and-place task is only complete if the robot grasps the object, lifts it without dropping it, and deposits it at the correct location --- in that order. The critical problem is not only individual step failure, but how the execution system responds after the failure. One failure mode is propagation: a robot that continues executing as if a failed grasp had succeeded will attempt to lift and place an object it is not holding. Later actions become meaningless, and the episode is lost. Another failure mode is stagnation: the policy may keep trying to grasp the same orange until the episode timeout, even when that orange is not being reached or grasped successfully and other oranges remain available.

Learned manipulation policies are particularly vulnerable to this failure propagation problem. They are trained on successful demonstrations, so their training distribution mostly contains states in which previous steps went well. During autonomous execution, small deviations move the robot into underrepresented states, yet the policy continues producing plausible-looking actions. In long-horizon tasks, these errors accumulate. Vision-Language-Action (VLA) models add language conditioning as an interface for specifying desired behaviours, but language alone does not solve the problem. An instruction such as ``place the object in the plate'' assumes the object is already held; if that assumption is false, the instruction offers no mechanism to interrupt or correct execution.

This project studies how to enable recovery in VLA-based robotic manipulation. The central hypothesis is that long-horizon tasks can be made more robust by decomposing them into language-conditioned subtasks and surrounding the policy with an orchestrator that monitors whether each subtask achieved its intended physical outcome. When a subtask fails, the orchestrator can retry the same sub-goal --- the core recovery mechanism studied here --- or redirect execution toward a feasible alternative to preserve overall task progress. This hypothesis is studied in a simulated kitchen manipulation task where an SO-101 arm must pick three oranges and place them into a plate.

The contributions of this work are threefold. First, it demonstrates that language conditioning brings no benefit in a monolithic execution setting, motivating the subtask decomposition approach. Second, it constructs a language-conditioned subtask dataset with randomised pick orders and implements an orchestrator that monitors subtask outcomes, enables local retry, performs scripted spatial reset before target changes, and redirects execution away from repeatedly failing targets. Third, it uses the orchestrated execution system as a basis for autonomous data generation, filtering for successful subtask trajectories to support future policy improvement.

The remainder of this report is structured as follows. \Cref{sec:related} reviews relevant work on learned manipulation, language-conditioned policies, and failure recovery. \Cref{sec:methods} describes the simulation environment, dataset construction, policy training, and orchestrator design. \Cref{sec:experiments} reports the experimental results. Finally, \Cref{sec:discussion} discusses implications and future work.

\section{Discussion \& Conclusion}
\label{sec:discussion}

The results so far support the central motivation of this project: improving long-horizon manipulation robustness cannot be reduced to changing the policy architecture alone. In the full-episode baseline, ACT and SmolVLA achieve comparable performance when both are trained on the same monolithic demonstrations and executed without recovery. This indicates that the main limitation is not only the expressiveness of the low-level policy, but also the structure of the execution problem.

The monolithic formulation gives the policy no explicit opportunity to verify that an intermediate physical outcome has been achieved. If a local subtask fails, the policy can continue producing plausible actions even though the assumptions needed for those actions are false. For a long-horizon task, this creates a compounding failure problem: a small error early in the sequence can invalidate the rest of the episode.

Subtask decomposition addresses this issue by introducing intervention points. A grasp, lift, placement, or return-to-start behaviour can be evaluated before the system commits to the next step. This does not make the low-level policy perfect, but it changes how failures affect the global task. Instead of silently propagating through the remainder of the episode, a failure can be detected and handled by the execution system.

The sliced public dataset experiment shows that subtask boundaries are not sufficient by themselves. For recovery, the system must also be able to choose among alternative valid continuations. In the orange-picking task, this means being able to request a specific target orange. A dataset with a fixed object order does not provide the right supervision for this behaviour, because target identity is entangled with progress through the task. The manually generated dataset was therefore necessary to make relative-position instructions meaningful.

The current recovery mechanism should be understood as task-level recovery rather than full correction of every local failure. Redirecting to a different orange does not recover the original target-specific objective. However, it can recover progress toward the global objective of placing all oranges into the plate. This distinction is important: the system is not yet solving arbitrary recovery from all possible failure states, but it demonstrates a practical way to avoid dead-end execution in a structured manipulation task.

Several limitations remain. First, the success checks use privileged simulator state. This is useful for isolating the recovery problem, but it would need to be replaced or approximated by perception-based success detection for deployment outside simulation. Third, autonomous data generation is still ongoing, so the effect of retraining on generated subtasks has not yet been measured.

A further extension would replace privileged success checks with visual or multimodal failure detection, making the recovery framework less dependent on simulator-only information.

In conclusion, this project frames recovery as an execution-level requirement for VLA-based robotic manipulation. The work so far shows that monolithic full-episode imitation has limited robustness, that subtask decomposition creates the intervention points needed for recovery, and that target-conditioned subtasks are necessary for meaningful redirection after failure. The ongoing autonomous data generation phase aims to use this recovery-aware execution structure not only to improve task completion, but also to collect better data for future policy improvement.

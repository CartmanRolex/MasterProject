\section{Introduction}
\label{sec:introduction}

Long-horizon robotic manipulation requires executing a sequence of physical interactions whose success depends on the evolving state of the world. A pick-and-place task is only complete if the robot grasps the object, lifts it without dropping it, and deposits it at the correct location --- in that order. The critical problem is not only individual step failure, but how the execution system responds after the failure. One failure mode is stagnation: the policy may keep trying to grasp the same orange until the episode timeout, even when that orange is not being reached or grasped successfully and other oranges remain available. A second, more speculative risk is that a failure leaves the scene in a state the policy has rarely seen --- a missed or fumbled grasp can knock an orange into an unusual position, and from such an out-of-distribution state the policy may act unreliably. We treat this state-poisoning effect as a hypothesis rather than an established mechanism.

Learned manipulation policies are particularly vulnerable to this kind of failure. They are trained on successful demonstrations, so their training distribution mostly contains states in which previous steps went well. During autonomous execution, small deviations move the robot into underrepresented states, yet the policy continues producing plausible-looking actions. In long-horizon tasks, these errors accumulate. Vision-Language-Action (VLA) models add language conditioning as an interface for specifying desired behaviours, but language alone does not solve the problem. An instruction such as ``place the object in the plate'' assumes the object is already held; if that assumption is false, the instruction offers no mechanism to interrupt or correct execution.

This project studies whether recovery can be enabled in VLA-based robotic manipulation by changing the execution structure around the policy. The hypothesis is that long-horizon tasks might become more robust if they are decomposed into language-conditioned subtasks and surrounded by an orchestrator that monitors whether each subtask achieved its intended physical outcome. When a subtask fails, the orchestrator can recover at two levels: it can retry the same sub-goal (recovering the specific orange), or redirect execution toward a feasible alternative orange (recovering overall task progress). Both levels of recovery are studied here. This hypothesis is tested in a simulated kitchen manipulation task where an SO-101 arm must pick three oranges and place them into a plate.

The contributions of this work are threefold. First, it compares ACT and SmolVLA in a monolithic full-task setting and shows that changing the policy class alone is not enough to solve the task reliably. Second, it constructs a language-conditioned Teleop subtask dataset with randomised pick orders and implements an orchestrator that monitors subtask outcomes, enables local retry, performs a controller-driven home-pose reset before target changes, and redirects execution away from a target after a GRASP timeout. Third, it uses the orchestrated execution system as a basis for autonomous data generation and evaluates whether the added successful subtasks improve the learned policy. The result is a useful negative finding: under the current seeded evaluation protocol, the added autonomous data reduces the orchestrated subtask performance.

The remainder of this report is structured as follows. \Cref{sec:methods} describes the simulation environment, dataset construction, policy training, and orchestrator design. \Cref{sec:experiments} reports the experimental results. Finally, \Cref{sec:discussion} discusses implications and future work.
